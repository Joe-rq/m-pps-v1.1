## 什么是 Claude Skills

**Claude Skills** 是 Claude AI 的一种扩展机制，允许你为 Claude 添加专门的能力和知识领域。通过上传 Skill 文件（通常是 Markdown 格式），Claude 可以：

- 理解特定领域的概念和规范
- 执行结构化的工作流程
- 维护长期的上下文和状态
- 与外部数据格式（如 JSON）交互

**M-PPS 系统**正是基于 Claude Skills 构建的个人操作系统。

```
你的自然语言输入

       ↓
Structure DNA 结构化
       ↓
JSON Ledger 存储
```

3. **状态机** - 统一的生命周期管理
4. **能量感知** - 根据你的精力周期智能调度

## 如何导入 Claude Skills

### 方法一：通过 Claude 界面导入（推荐）

1. **打开 Claude 对话**
2. **点击 "+" 或 "添加" 按钮**（通常在输入框附近）
3. **选择 "Add Skill" 或 "添加技能"**
4. **上传 Skill 文件**：
   - 导航到 `skills/goal-manager/SKILL.md`
   - 选择并上传该文件
5. **确认导入** - Claude 会显示 Skill 已加载
6. **重复以上步骤**导入其他三个 Skills

### 方法二：通过复制粘贴

如果界面不支持文件上传：

1. **打开 Skill 文件**（如 `skills/goal-manager/SKILL.md`）
2. **复制全部内容**
3. **在 Claude 对话中发送**：
   ```
   请加载以下 Skill 定义：
   [粘贴 Skill 文件内容]
   ```
4. Claude 会确认已理解该 Skill

### 导入顺序建议

```
1. ledger-registry (账本注册器) - 基础设施
2. goal-manager (目标管理器) - 创建目标
3. personal-schedule-manager (日程管理器) - 安排时间
4. reflection-manager (反思管理器) - 记录学习
```

## 四个核心 Skills 详解

### 1️⃣ goal-manager（目标管理器）

**作用**：将你的自然语言意图转化为结构化的目标条目

**使用方式**：
```
# 对 Claude 说：
"我想学习 Python 数据分析，3个月内掌握基础"

# Claude 会：
- 解析你的意图
- 创建 G-001 目标条目
- 写入 ledger/goal.json
- 返回确认信息
```

**输出示例**：
```json
{
  "id": "G-001",
  "title": "学习 Python 数据分析",
  "status": "open",
  "estimated_duration": "3 months",
  "tags": ["learning", "python", "data-analysis"]
}
```

**关键命令**：
- "创建一个新目标：..."
- "添加目标：..."
- "我想要..."

---

### 2️⃣ personal-schedule-manager（日程管理器）

**作用**：根据目标和你的能量周期生成可执行的日程安排

**使用方式**：
```
# 对 Claude 说：
"为 G-001 生成本周的学习计划"

# Claude 会：
- 读取 profile.json 了解你的能量周期
- 在你的高效时段安排学习任务
- 创建 S-001, S-002... 日程条目
- 写入 ledger/schedule.json
```

**智能调度特性**：
- **能量匹配**：创造性任务安排在晚间高峰期
- **时间块优化**：根据你的偏好分配时间长度
- **避开低能量期**：简单任务放在精力较低时段

**关键命令**：
- "安排本周的学习计划"
- "为这个目标创建日程"
- "什么时候学习最好？"

---

### 3️⃣ reflection-manager（反思管理器）

**作用**：记录任务完成情况、学习收获和改进建议

**使用方式**：
```
# 对 Claude 说：
"今天完成了 S-001，学习了 pandas 基础，感觉理解了 DataFrame 概念"

# Claude 会：
- 更新 S-001 状态为 done
- 创建 R-001 反思条目
- 记录学习收获
- 更新能量匹配度
- 写入 ledger/reflection.json
```

**反思内容包括**：
- 任务完成情况
- 学习收获与困难
- 能量匹配度（实际精力是否符合预期）
- 改进建议

**关键命令**：
- "完成了 [任务]，今天学到了..."
- "标记 S-001 为完成"
- "生成今日反思"

---

### 4️⃣ ledger-registry（账本注册器）

**作用**：维护所有账本文件的完整性和一致性

**使用方式**：
```
# 对 Claude 说：
"检查账本系统状态"
"验证所有 ledger 文件"

# Claude 会：
- 读取 manifest.json
- 验证所有账本文件存在
- 检查数据格式正确性
- 更新 checksum
- 报告系统健康状态
```

**维护功能**：
- 自动创建缺失的账本文件
- 验证数据完整性
- 生成系统报告

**关键命令**：
- "检查系统状态"
- "验证账本"
- "初始化 ledger 系统"

## 常见问题

### Q1: Skills 导入后不生效？

**答**：确保：
- 完整复制了 Skill 文件内容
- Claude 确认已理解该 Skill
- 尝试重新开启对话

---

### Q2: JSON 文件在哪里？

**答**：
- 所有账本文件在 `ledger/` 目录下
- 使用文本编辑器（VS Code、Notepad++）可以查看
- Claude 会自动读写这些文件

---

### Q3: 如何修改能量周期设置？

**答**：
1. 打开 `ledger/profile.json`
2. 修改 `energy_curve` 部分
3. 调整 `time` 字段为你的实际情况
4. 保存文件
5. 对 Claude 说"重新读取 profile 配置"

---

### Q4: 可以自己创建新的 Skill 吗？

**答**：
可以！参考现有 Skills 的格式：
1. 定义 Skill 的目标和功能
2. 说明输入输出格式
3. 编写伪代码或工作流程
4. 保存为 `.md` 文件
5. 导入到 Claude

---

### Q5: 如何查看系统状态？

**答**：
```
# 对 Claude 说：
"显示系统状态"
"有多少个活跃目标？"
"本周有哪些计划？"
"最近的反思是什么？"
```

---

### Q6: 时间格式是什么？

**答**：
使用 **ISO-8601** 格式：
- `2025-11-15T14:00:00+08:00`
- 年-月-日T时:分:秒+时区

Claude 会自动处理时区转换。

---

### Q7: 如何备份数据？

**答**：
1. 复制整个 `ledger/` 目录
2. 或使用版本控制（Git）
3. 定期导出为 JSON 备份
